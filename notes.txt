// Convensions
* Each UI View (folders inside UI folder) has a corresponding controller (folders inside Controllers folder) . There could be some UI view that does not need a controller (AuthenticationView)
* One UI view is made of UI components (files inside each view folder), these components are in charge in components inside a view, ejem (panel assets in the main view AnimationView). Each view is composed by files, each file is a component. There is a main component (file), whose main class represents all the components, means to be the view. This class has direct access to the controller associated to the view
* In turn each UI component can have sections (clases inside one file), these are sections with enough complexity to be extracted into a different class. But there will be a main class which is the entire component itself
* In a UI view should reside only UI logic, and in the controller the business logic
